Conformance Results

Browsers used
  • Chrome Version 41.0.2272.76 (64-bit)
  • Firefox 36.0 (Mozilla Firefox for Ubuntu canonical -1.0)
  • Internet Explorer 11.0.09600.17728
SECTION 4.4-4.5
  • No difference between firefox and Chrome and IE.
SECTION 4.6-4.7
  • Object of type mp4 does not display in IE.
  • Autoplay on/off works in Firefox only.
  • Video controls not displayed in Firefox and cannot be toggled in chrome
  • <video> works only in IE.
  • <audio> source not found in IE.
SECTION 4.8-4.9
  • Firefox does not display the lines that make up the table
  • Otherwise, IE, Firefox, and Chrome look identical
SECTION 4.10-4.14
  • IE draws gray instead of black lines around sections. This makes it much harder to see the boundary lines and makes things look a little more confusing.
  • For displaying encryption levels, Chrome will show a number along with each level while firefox will simplify it to just "High Grade" or "Low Grade". IE does not display any dropdown or options for the security level since it is not supported.
  • For readonly sections, Firefox grays out the text area, making it a lot more obvious that it is a read-only field. IE and Chrome, however, do not do this so the text area looks as if it already has data entered.
  • For the Age/number element, Firefox shows the up and down arrows which makes it clear that you can increment or decrement the number value using the arrows. Chrome only displays the arrows when placing the pointer over the textbox. IE does not display this arrow and the user cannot utilize this to manipulate the value in the textbox.
  • In general, the visuals are different between each browser. Chrome and IE look more boxy while firefox is is cleaner and uses colors to indicate states more often. Also, Firefox uses more orange color while IE uses some blue with mostly black and white variations. Chrome uses more blue color for certain elements.
  • For the happiness meter (range element), IE will show the value as the slider is dragged. This makes it clear to the user what value they are selecting. Chrome and Firefox, however, do not do this.
  • The date formats are different in each browser. Firefox use a format of YYYY-MM-DD to represent the date while chrome uses MM/DD/YYYY. Other date related inputs, such as the Week input, is also different. Chrome displays weeks as "Week 01, 1990" while firefox displays "1990-W01". IE uses the same formatting as Firefox.
  • IE does not open a color picker when clicking the "color input" option. Instead, IE has a text box in which a hex value needs to be inputed.
  • IE does not support the display of meters. It will instead display the text value of the meter, such as 75% or 2/10.
  • IE was unable to utilize javascript to draw a rectangle on a canvas.
